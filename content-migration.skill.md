# content-migration Skill

## 负责什么

负责将旧官网内容资产迁移到新官网的信息架构中，包括旧官网页面审计、产品资料提取、参数清洗、案例归类、公司介绍整理、联系方式核对、图片视频素材盘点、文案重写和内容去重。

本 Skill 只处理内容资产，不继承旧官网的视觉风格、页面布局、组件样式、交互方式和 CMS 模板结构。

## 必须坚持什么

- 旧官网只作为内容来源，不作为设计参考。
- 保留真实产品、真实参数、真实案例、真实联系方式、真实企业资质和真实作业素材。
- 所有产品参数必须尽量保持原始数值、单位和含义，不得为了视觉效果擅自夸大。
- 所有案例必须保留关键事实：时间、地点、产品型号、任务、作业效果、可量化数据。
- 内容迁移必须以“国产高端低空经济品牌官网”为目标重新组织。
- 首页只呈现高价值内容，不照搬旧站栏目。
- 产品、场景、案例、公司、联系信息必须建立清晰映射关系。
- 旧文案中可用事实和不可用表达要分离处理。
- 对疑似错误、重复、过时或冲突的信息必须标记为“待确认”。
- 地址、产品命名、历史型号、案例时间等关键信息不得含糊处理。

## 禁止做什么

- 禁止复制旧官网整段营销文案作为新版正式文案。
- 禁止继承旧官网的轮播、侧栏、面包屑、推荐新闻、产品列表模板等结构。
- 禁止保留“欢迎访问产品列表页”“立即购买”“热门产品”“高性价比”等模板站话术。
- 禁止把法规全文、新闻转载、重复动态直接堆进新版页面。
- 禁止删除产品参数中的关键能力数据。
- 禁止用新概念掩盖旧内容中的真实缺口。
- 禁止把所有案例平铺成新闻列表。
- 禁止把图片素材按旧站裁切比例和排版方式直接复用。
- 禁止把旧站的低质占位文案迁移到任何新页面。

## 如何检查质量

- 是否完成页面清单、产品清单、参数表、案例清单、素材清单和联系方式清单。
- 是否区分“必须迁移”“需要重写”“可以合并”“可以删除”。
- 是否为每个产品建立了定位、参数、场景、案例和素材关联。
- 是否把新闻内容转化为品牌背书、场景案例或政策背景，而不是照搬新闻流。
- 是否标记了冲突信息，例如地址版本、产品型号命名、重复案例。
- 是否能支持后续首页、产品页、场景页、案例页、关于页和联系页的信息架构。
- 是否所有可迁移内容都服务于高端低空经济品牌定位。

## 给后续代码生成的具体约束

- 代码生成前必须先建立内容迁移表，字段至少包括：来源页面、原始内容、内容类型、新站位置、处理方式、可信度、待确认事项。
- 产品参数必须进入结构化数据，不得只写死在大段文案里。
- 案例必须进入可复用数据结构，字段至少包括：标题、产品、场景、地点、时间、任务、结果、关键数据、素材。
- 联系方式必须集中管理，不得在多个组件中重复硬编码。
- 旧站素材必须建立素材库命名规范，不得以旧 CMS 文件名直接散落使用。
- 首页文案、产品文案和 CTA 文案必须基于清洗后的内容重写。
- 所有页面必须能追溯其内容来源，方便后续校对。
