# luxury-ui-system Skill

## 负责什么

负责新官网的高级 UI 系统，包括字体、色彩、排版、间距、组件、按钮、导航、数据展示、表单、响应式规则和设计系统一致性。

本 Skill 确保网站具有高端品牌、工业装备和发布页级别的界面质感。

## 必须坚持什么

- UI 必须克制、精密、清晰、耐看。
- 字体层级必须强，标题有品牌气势，正文有专业可读性。
- 色彩必须高级，不使用廉价科技蓝作为主调。
- 主色建议围绕石墨黑、暖白、金属灰、深军绿、低饱和安全橙或冷金属色建立。
- 组件必须少而精，避免卡片泛滥。
- 数据展示必须像高端装备参数，而不是 SaaS 面板。
- 按钮、标签、参数、导航必须统一设计语言。
- 移动端必须保持高级感，不得只是桌面端压缩。
- 表单必须显得专业可信，适合高价值询盘。
- 图文比例必须服务内容优先级，不能为了布局牺牲产品主体。

## 禁止做什么

- 禁止使用模板站常见大圆角卡片堆叠。
- 禁止使用过度渐变、玻璃拟态、彩色阴影、霓虹按钮。
- 禁止把页面做成 SaaS 仪表盘。
- 禁止大量使用图标墙代替真实内容。
- 禁止文字贴边、间距拥挤、按钮尺寸混乱。
- 禁止字体过多、字号层级混乱。
- 禁止在工业高端官网中使用幼稚插画和卡通图标。
- 禁止页面主色被单一蓝色或蓝紫色统治。
- 禁止文字压在复杂图片上导致不可读。

## 如何检查质量

- 页面截图是否像高端装备品牌，而不是建站模板。
- 色彩是否克制且有品牌识别度。
- 字号、行距、间距是否稳定。
- CTA 是否清晰但不廉价。
- 参数表是否专业、易读、统一。
- 导航是否简洁，能承载品牌站和产品站双重需求。
- 移动端是否仍然保留留白和秩序。
- 组件是否服务内容，而不是为了装饰存在。

## 给后续代码生成的具体约束

- 必须先定义设计令牌，包括颜色、字体、字号、间距、圆角、阴影、边框、动效时长。
- 全站按钮必须使用统一组件，不得各处单独写样式。
- 数据指标、参数表、产品卡片、案例卡片和表单必须有统一组件规范。
- 卡片圆角默认不超过 8px，除非设计系统明确要求。
- 不得嵌套卡片。
- 不得使用 viewport 宽度驱动字号。
- 所有固定格式 UI 必须有稳定尺寸或响应式约束，避免 hover、加载和动态内容造成布局跳动。
- 文本必须在移动端和桌面端都不溢出、不遮挡、不重叠。
