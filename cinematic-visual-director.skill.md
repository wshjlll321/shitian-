# cinematic-visual-director Skill

## 负责什么

负责新官网的电影感视觉导演，包括镜头语言、画面节奏、光影关系、留白比例、滚动叙事、视频素材使用、画面裁切和情绪控制。

本 Skill 确保官网不是信息堆叠页，而是具有发布会级叙事节奏和高端产品影像感的品牌体验。

## 必须坚持什么

- 每个关键页面都必须有清晰的镜头顺序：建立场景、突出产品、展示任务、给出数据、完成转化。
- 大图和视频必须服务叙事，不做装饰背景。
- 画面应体现低空、农田、山林、海面、救援现场、工业细节和产品结构。
- 留白必须足够，让产品和数据有呼吸感。
- 光影应克制高级，偏真实摄影质感，不追求过度发光。
- 页面节奏要有起伏：大画面、短文案、数据、场景、细节、案例交替出现。
- 产品出现时要像高端装备发布，而不是商品陈列。
- 案例出现时要像纪录片片段，而不是新闻列表。
- 动效必须模拟镜头推进、景深切换、信息显影、视线引导。

## 禁止做什么

- 禁止把视频或图片只当成铺满背景的装饰。
- 禁止每屏都使用同一种卡片网格。
- 禁止过度暗黑、过度蓝光、过度镜头眩光。
- 禁止用假 3D、假 HUD、假雷达界面制造科技感。
- 禁止密集堆叠文字破坏画面。
- 禁止使用随机飞入、弹跳、旋转、闪烁等廉价动效。
- 禁止用 AI 抽象图替代真实场景。
- 禁止为了炫技让页面滚动失控或影响阅读。
- 禁止把所有内容做成横向滑动长廊，除非它确实服务产品叙事。

## 如何检查质量

- 每一屏是否可以被描述为一个明确镜头。
- 首屏是否有品牌压迫感和产品存在感。
- 画面是否足够干净，重点是否清楚。
- 滚动过程中是否有叙事推进，而不是模块堆叠。
- 产品、场景、数据之间是否形成节奏。
- 动效是否让用户更理解产品能力。
- 图片裁切是否保留产品主体和任务现场，不遮挡关键信息。
- 页面是否在移动端仍然有镜头感，而不是变成信息碎片。

## 给后续代码生成的具体约束

- Hero 区必须预留真实产品或真实场景媒体位，不得只使用渐变背景。
- 关键页面必须规划媒体层、文案层和数据层的前后关系。
- 滚动动效必须有最大复杂度限制，不能影响首屏加载和核心内容可读性。
- 视频必须设置封面、降级图和移动端处理策略。
- 视觉模块必须有稳定宽高比，避免加载时布局跳动。
- 文案不得覆盖产品主体、人物、二维码、关键作业画面。
- 每个大画面模块必须有明确叙事目的，例如“建立产品尺寸感”“证明作业效率”“展示复杂地形”。
