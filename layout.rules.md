# Layout Rules

## Rule Intent

These rules control page structure, spatial rhythm, section variation, responsive layout, and content hierarchy.

## Mandatory Rules

- Layout must create cinematic page rhythm.
- Sections must vary in height, density, composition, and media ratio.
- The homepage must feel like a premium product and scenario narrative, not a list of equal modules.
- Use large visual moments, quiet text moments, specification moments, case moments, and conversion moments.
- Product visuals must be large enough to create physical presence.
- Scenarios must be expressed through real tasks, not simple icon grids.
- Layout must guide users from brand authority to product capability to proof to inquiry.
- Mobile layout must be designed intentionally, not merely stacked.
- Each page must have clear primary, secondary, and tertiary content zones.
- Whitespace must be structural, not leftover empty space.

## Forbidden Rules

- Do not use ordinary three-column card stacking as the main layout.
- Do not make all sections the same height.
- Do not make every screen "heading + paragraph + button".
- Do not create nested cards.
- Do not place UI cards inside decorative page-section cards.
- Do not use generic icon walls for application scenarios.
- Do not repeat the same grid pattern across the whole site.
- Do not make the homepage a sequence of equal blocks.
- Do not let parameter tables dominate the first impression.
- Do not use old website navigation logic or CMS page order as new information architecture.

## Required Layout Rhythm

The homepage must include layout variation such as:

- Full-bleed hero with product or scenario presence
- Product focus section with asymmetric composition
- Metric strip or sparse technical proof
- Scenario narrative sections with real case anchors
- HIZONE system explanation with process or closed-loop structure
- Case proof section with varied media weights
- Professional inquiry section

Product pages must include:

- Product identity hero
- Capability narrative
- Scenario proof
- Specification module
- Media proof
- Related inquiry CTA

## Failure Conditions

Layout fails review if:

- It looks like a template.
- It could be rearranged without changing meaning.
- It relies on repeated cards instead of narrative hierarchy.
- It gives all products equal visual weight despite different strategic importance.
- It presents application scenarios as icons without proof.
- It has no cinematic pauses.
- It has no clear conversion path.

## Generation Constraints

- Define page information architecture before building components.
- Define section purpose before section layout.
- Use responsive constraints, aspect ratios, and max widths deliberately.
- Ensure text never overlaps other text or critical media.
- Ensure navigation, CTA, and contact paths remain visible or easily reachable.
- Use cards only for repeated items, compact records, forms, or modals.
